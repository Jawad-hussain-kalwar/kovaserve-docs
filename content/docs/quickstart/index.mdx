**KovaServe is the Agentic Execution Cloud.** It helps neoclouds, sovereign AI platforms, and enterprises launch managed agents that run durably, recover from failure, operate under governance, and control cost across private infrastructure. The unit of value is a **completed agent run** — not a token, not a request.

This Quickstart walks you from a fresh machine to a completed managed agent run in five minutes.

## What you'll have after this Quickstart

- One completed managed agent run, with a full timeline of steps, model calls, and tool actions.
- A clear view of cost per completed run — the metric that matters.
- A budget attached to the run, with warning and hard-stop thresholds enforced.
- An execution audit trail you can replay end-to-end.

## What KovaServe does

KovaServe gives production agents:

- **Durable execution** — agent runs survive interruptions, failures, pauses, and resumes.
- **Context and state control** — bounded, recoverable, governed context across long-running runs.
- **Checkpoint recovery** — important execution points saved so failures don't restart from zero.
- **Budget enforcement** — runs constrained by cost, time, steps, or tool count with active admission control.
- **Tool governance** — actions allowed, restricted, or forced through policy before execution.
- **Execution audit** — every model call, tool call, policy decision, and outcome recorded.
- **Cost per completed run** — economics that match the business outcome.

What KovaServe is **not**: an inference layer (it works with vLLM, llm-d, and managed model APIs), an agent framework (it sits beneath LangGraph, Pydantic AI, OpenAI Agents SDK, Temporal, and custom runtimes), or a context engineering platform (the runtime decides what goes in the prompt; KovaServe decides what survives, what compacts, what restores, and what counts against budget).
