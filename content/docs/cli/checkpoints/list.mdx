---
title: "`kova checkpoints list`"
description: List Checkpoints scoped to a Run or Project. Filter by status, trigger label, and time window.
---

List [Checkpoints](/docs/concepts/recovery/checkpoints) for a Run or across a Project.

## Synopsis

```bash
kova checkpoints list [OPTIONS]
```

## Options

| Option | Meaning |
|---|---|
| `--run-id <id>` | List Checkpoints for one Run. The common case. |
| `--project-id <id>` | Broader Project scope. Defaults to the active profile's Project when `--run-id` is omitted. |
| `--status <status>` | Filter by `CheckpointStatus`. Accept-multiple: `--status VALIDATED --status STORED`. |
| `--trigger <label>` | Filter by trigger label — `before_file_write`, `before_shell_command`, `after_stable_analysis`, `on_budget_warning`, `periodic`, `manual`, or any string. |
| `--limit <n>` | Per-page count. Default 50, max 100. |
| `--order <asc\|desc>` | Sort by `created_at`. Default `asc` (oldest first when scoping to a Run). |
| `--all` | Fetch every page. |
| `--json` | Emit a JSON array. |

## Examples

```bash
# All Checkpoints on a Run, oldest first
kova checkpoints list --run-id 01J8X9Y2Z3K4M5N6P7Q8R9S0T1

# Only restorable Checkpoints
kova checkpoints list --run-id 01J... --status VALIDATED

# Most recent Checkpoint for the Run
kova checkpoints list --run-id 01J... --order desc --limit 1

# All `before_file_write` Checkpoints across the Project
kova checkpoints list --trigger before_file_write
```
