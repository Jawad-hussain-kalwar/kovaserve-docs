Restore a [Run](/docs/concepts/run-hierarchy/runs) from a [Checkpoint](/docs/concepts/recovery/checkpoints). Logical-context-first; KV restore is layered on when available.

## Synopsis

```bash
kova checkpoints restore <CHECKPOINT_ID> [OPTIONS]
```

## Options

| Option | Meaning |
|---|---|
| `--mode <mode>` | One of `cold`, `warm-logical` (default), `warm-kv`. See modes table below. |
| `--target-node-id <id>` | Advisory placement hint for the resumed Run. |
| `--json` | Emit the restored Run as one JSON document. |

## Modes

| Mode | What restores | When to use |
|---|---|---|
| `cold` | Logical context only â€” no KV restore even if available. | Forcing a cold restart; debugging restore correctness. |
| `warm-logical` (default) | Logical context + KV restore if the `kv_prefix` StateHandle is resident; falls back to logical-only if KV restore fails. | The general-purpose path. |
| `warm-kv` | Logical context + KV restore â€” fails the restore if the `kv_prefix` StateHandle is unavailable. | Hard requirement on a warm restart; next operation only makes sense against a warm cache. |

## Examples

```bash
# Default â€” warm-logical restore
kova checkpoints restore 01J8X9Y2Z3K4M5N6P7Q8R9S0T7

# Force a cold restart for debugging
kova checkpoints restore 01J... --mode cold

# Require a warm restart (fails if KV unavailable)
kova checkpoints restore 01J... --mode warm-kv

# JSON for scripting
kova checkpoints restore 01J... --json | jq '.status'
```

## Failure modes

| Error | Cause | Suggested fix |
|---|---|---|
| `400 model_incompatible` | The Checkpoint's `model_family` / `tokenizer_fingerprint` / `renderer_version` doesn't match any registered model. | Roll back the model swap or restore to a compatible cluster. |
| `400 kv_incompatible` | KV restore plan fails on `quantization` / `tensor_parallel` mismatch. | Use `--mode warm-logical` to skip KV restore. |
| `410 kv_state_unavailable` | `--mode warm-kv` requested but the `kv_prefix` StateHandle is evicted. | Re-issue with `--mode warm-logical`. |
| `404 artifact_not_found` | A referenced ContextArtifact has been deleted or evicted. | Check artifact retention policy; the Checkpoint may be unrestorable. |
