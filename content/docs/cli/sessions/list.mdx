---
title: "`kova sessions list`"
description: List Sessions filtered by state, last-accessed time, or Project. Cursor-paginated; `--json` emits the page as one JSON array.
---

List [Sessions](/docs/concepts/continuity/sessions) in the active Project.

## Synopsis

```bash
kova sessions list [OPTIONS]
```

## Options

| Option | Meaning |
|---|---|
| `--project-id <id>` | Defaults to the active profile's Project. |
| `--state <state>` | Filter by `SessionState`. Accept-multiple: `--state OPEN --state IDLE`. |
| `--last-accessed-after <iso8601>` | Filter to Sessions accessed on or after this timestamp. |
| `--created-after <iso8601>` / `--created-before <iso8601>` | Time window on creation. |
| `--limit <n>` | Per-page count. Default 50, max 100. |
| `--order <asc\|desc>` | Sort by `last_accessed_at`. Default `desc`. |
| `--all` | Fetch every page. |
| `--json` | Emit a JSON array. |

## Examples

```bash
# Active Sessions in the active Project
kova sessions list --state OPEN

# Idle Sessions older than a week — candidates for archive
kova sessions list --state IDLE --last-accessed-after $(date -u -d "-7 days" +%Y-%m-%dT%H:%M:%SZ)

# JSON for scripting
kova sessions list --json | jq '.[] | {id: .id, turns: .turn_count}'
```
