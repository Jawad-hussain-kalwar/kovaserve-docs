---
title: "`kova sessions get`"
description: Fetch one Session by ID, optionally with turn history and the current Run hydrated inline.
---

Fetch one [Session](/docs/concepts/continuity/sessions) by ID. Session IDs follow the `ses_<24hex>` format.

## Synopsis

```bash
kova sessions get <SESSION_ID> [OPTIONS]
```

## Options

| Option | Meaning |
|---|---|
| `--expand <path>` | Hydrate nested resources. Valid paths: `turns`, `current_run`, `current_manifest`. Repeat for multiple. |
| `--limit-turns <n>` | When `--expand turns` is set, cap the number of turns returned. Default unbounded. |
| `--json` | Emit the Session as one JSON document. |

## Examples

```bash
# Bare Session row
kova sessions get ses_a7f3c2b8d4e1f0925a1b6c3d

# With turn history
kova sessions get ses_a7f3c2b8d4e1f0925a1b6c3d --expand turns

# With turn history and current Run details
kova sessions get ses_a7f3c2b8d4e1f0925a1b6c3d --expand turns --expand current_run

# JSON for scripting
kova sessions get ses_a7f3c2b8d4e1f0925a1b6c3d --expand turns --json | jq '.turns[].role'
```
