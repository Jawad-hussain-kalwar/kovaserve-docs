---
title: "`kova runs get`"
description: >-
  Fetch one Run by ID, with optional `--expand` paths to hydrate Steps, ModelCalls, and ContextManifests in one call.
---

Fetch one [Run](/docs/concepts/run-hierarchy/runs) by ID. Renders as a tree by default; use `--expand` to hydrate nested resources.

## Synopsis

```bash
kova runs get <RUN_ID> [OPTIONS]
```

## Options

| Option | Meaning |
|---|---|
| `--expand <path>` | Dotted path to hydrate. Repeat for multiple paths: `--expand steps --expand steps.model_calls`. |
| `--flat` | Render Steps as a flat ordinal-ordered list instead of a tree. |
| `--max-depth <n>` | Cap tree depth. Default 5. Pass 0 for unlimited. |
| `--json` | Emit the full Run as one JSON document. |

## Examples

```bash
# Basic — bare Run row
kova runs get 01J8X9Y2Z3K4M5N6P7Q8R9S0T1

# With Steps and ModelCalls
kova runs get 01J8X9Y2Z3K4M5N6P7Q8R9S0T1 --expand steps.model_calls

# With full graph hydration
kova runs get 01J... --expand steps.model_calls.context_manifest

# JSON for scripting
kova runs get 01J... --expand steps --json | jq '.steps[].step_type'

# Flat ordinal list
kova runs get 01J... --expand steps --flat
```
