List and inspect [ContextManifests](/docs/concepts/context/context-manifests).

## Subcommands

| Subcommand | What it does |
|---|---|
| `kova context manifests list` | Cursor-paginated list scoped to a Run, Session, or Project. |
| `kova context manifests get <ID>` | Fetch one manifest. `--expand segments` hydrates the segment list. |

## Examples

```bash
# Every manifest for a Run, oldest first
kova context manifests list --run-id 01J8X9Y2Z3K4M5N6P7Q8R9S0T1

# Filter to specific admission decisions
kova context manifests list --run-id 01J... --decision DEGRADE --decision HARD_STOP

# Fetch a manifest with segments
kova context manifests get 01J8X9Y2Z3K4M5N6P7Q8R9SCM07 --expand segments

# Find all calls with one stable_context_id (cache-hit analysis)
kova context manifests list --stable-context-id sha256:7e44...
```

## `manifests list` options

| Option | Meaning |
|---|---|
| `--run-id <id>` | The common filter. |
| `--session-id <id>` | Filter to a Session. |
| `--project-id <id>` | Broader Project scope. |
| `--stable-context-id <hash>` | Find calls sharing a context identity. |
| `--decision <value>` | Filter by admission decision. Accept-multiple: `--decision WARN --decision DEGRADE`. |
| `--created-after <iso8601>` / `--created-before <iso8601>` | Time window. |
| `--limit <n>` | Per-page count. |
| `--order <asc\|desc>` | Sort by `created_at`. |
| `--all` | Fetch every page. |
| `--json` | Emit a JSON array. |

## `manifests get` options

| Option | Meaning |
|---|---|
| `--expand <path>` | Valid: `segments`, `model_call`. |
| `--json` | Emit the manifest as one JSON document. |
