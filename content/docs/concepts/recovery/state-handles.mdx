A **StateHandle** is a typed reference to reusable execution state. It generalizes the original "activation handle" concept so cross-Session reuse extends beyond raw KV cache prefixes to manifests, artifacts, and runtime-local state.

StateHandles are the connective tissue between the entities that *produce* reusable state ([ContextManifest](/docs/concepts/context/context-manifests), [ContextArtifact](/docs/concepts/context/context-artifacts), KV cache exports, runtime workspace snapshots) and the entities that *consume* it ([Sessions](/docs/concepts/continuity/sessions), [ModelCalls](/docs/concepts/run-hierarchy/model-calls), [Checkpoints](/docs/concepts/recovery/checkpoints)). The State Handle Registry is the system of record.

## The discriminator

Every StateHandle carries a `type` discriminator that determines what the `ref` field points at and how the Checkpoint Manager validates it.

| `type` | What it points at | Validator |
|---|---|---|
| `kv_prefix` | A KV cache prefix exported from an inference worker on the substrate. | Tokenizer fingerprint, model family, residency tier, freshness. |
| `context_manifest` | A frozen `ContextManifest` row. | Renderer version, model id, scope match. |
| `checkpoint` | Another Checkpoint (composition or chained restore). | Target Checkpoint must be `VALIDATED` or `RESTORED`. |
| `artifact_bundle` | A set of `ContextArtifact` rows. | Each artifact readable from object storage; hashes match. |
| `runtime_state` | Layer 3 runtime state (workspace snapshot, sandbox FS, process tree). | Sandbox image compatibility; workspace bind-mount layout. |

The discriminator set is closed â€” these five types are the entire surface. New types are added by the State Handle Registry through an explicit migration; runtimes do not invent local subtypes.

```json
{
  "id": "01J8X9Y2Z3K4M5N6P7Q8R9S0T9",
  "type": "kv_prefix",
  "ref": "kv://node-2/01J8X9Y2Z3K4M5N6P7Q8R9S0T7",
  "compatibility": {
    "model_family": "qwen3-coder",
    "tokenizer_fingerprint": "sha256:9f2câ€¦",
    "renderer_version": "v3"
  },
  "scope": {
    "tenant_id": "01J8X9Y2Z3K4M5N6P7Q8R9S0T0",
    "project_id": "01J8X9Y2Z3K4M5N6P7Q8R9S0T1"
  },
  "created_at": "2026-04-30T12:00:00Z"
}
```

`id` is a bare 26-character ULID. `ref` is a type-specific opaque string the registry knows how to resolve. `compatibility` is populated according to the discriminator (KV-shaped fields for `kv_prefix`, manifest-shaped fields for `context_manifest`, etc.).

## Resolving a handle

```http
GET /v1/ks/handles/01J8X9Y2Z3K4M5N6P7Q8R9S0T9
Authorization: Bearer ks_...
```

The State Handle Registry returns the resolved target along with the validation result:

```json
{
  "id": "01J8X9Y2Z3K4M5N6P7Q8R9S0T9",
  "type": "kv_prefix",
  "ref": "kv://node-2/01J8X9Y2Z3K4M5N6P7Q8R9S0T7",
  "resolution": {
    "status": "available",
    "warmness": "warm",
    "node_id": "01J8X9Y2Z3K4M5N6P7Q8R9N002",
    "compatibility": "exact"
  }
}
```

| `resolution.status` | Meaning |
|---|---|
| `available` | The target exists, is compatible with the current scope, and can be used. |
| `incompatible` | The target exists but its compatibility vector does not agree with the request. |
| `expired` | The target's TTL or retention has elapsed. |
| `not_found` | The target is gone. For `kv_prefix` this typically means evicted; for `artifact_bundle` it means object-storage delete. |

## Where handles are produced

| Producer | Handles created |
|---|---|
| Substrate KV export at Checkpoint capture | One `kv_prefix` per exported prefix. |
| Checkpoint capture | One `context_manifest`, plus zero or more `artifact_bundle`s, plus optional `runtime_state`. |
| Global Context Manager (manifest assembly) | One `context_manifest` per produced manifest. |
| Context Artifact Store on artifact creation | One `artifact_bundle` for runs that retain artifacts at Session scope. |
| Layer 3 workspace snapshot | One `runtime_state` per snapshot. |

## Where handles are consumed

| Consumer | How it uses handles |
|---|---|
| [Checkpoint](/docs/concepts/recovery/checkpoints) | Binds one `context_manifest`, optionally one `kv_prefix`, zero or more `artifact_bundle`s, optionally one `runtime_state`. |
| [Resume flow](/docs/concepts/recovery/resume-flow) | Validates each handle independently before restore; logical-context handles are required, KV is optional. |
| [Cache-aware router](/docs/concepts/routing/manifest-aware-routing) | Reads `kv_prefix` and `context_manifest` handles when scoring candidate replicas. |
| [Sessions](/docs/concepts/continuity/sessions) | Holds the active `kv_prefix` handle for the running ModelCall and references prior `context_manifest` handles via the [current manifest](/docs/concepts/continuity/current-manifest). |

## Compatibility validation

Every consumer that resolves a handle revalidates compatibility against its own scope. The discriminator decides which fields are checked:

- `kv_prefix` â€” model family, tokenizer fingerprint, residency tier, freshness; mismatches cause silent skip in favor of manifest replay.
- `context_manifest` â€” renderer version, model id, scope; mismatches reject restore outright.
- `artifact_bundle` â€” every contained artifact hash; mismatches reject the bundle.

The contract is uniform: a handle resolves cleanly, or it does not. There are no partial-success resolutions â€” partial or warmness signals are advisory metadata, not resolution states.

## Sharing

Each StateHandle carries a `sharing` flag that bounds which scopes may resolve it:

| `sharing` | Resolution rule |
|---|---|
| `private` | Only the producing Run / Session / API key may resolve. Cross-session and cross-project requests return `404`. |
| `shareable_within_project` | Any caller in the same `(tenant_id, project_id)` may resolve. Cross-project requests return `404`. |
| `shareable_within_tenant` | Any caller in the same `tenant_id` may resolve, across projects. Cross-tenant requests return `404`. |

Cross-tenant resolution is always `404` regardless of `sharing` â€” the tenant boundary is hard. A handle whose underlying state has been evicted resolves with `410 handle_expired`. Sharing is set at creation and may be promoted (narrower â†’ broader) via `POST /v1/ks/handles/{id}?action=share`; demotion is not supported because consumers may already hold cached references.

## Lifetime

StateHandles are not deleted when the entity they reference is deleted. The handle row persists with `resolution.status = "not_found"` so audits can answer "what was this Checkpoint pointing at?" even after the underlying KV prefix has been evicted from the substrate. Bulk handle eviction is a tenant-retention operation, not a casual cleanup.

## ID format

StateHandle IDs are bare 26-character Crockford ULIDs:

```
01J8X9Y2Z3K4M5N6P7Q8R9S0T9
```

This matches the convention for `run_id`, `step_id`, `model_call_id`, `checkpoint_id`, and `context_manifest_id`. Only `session_id` deviates; see [Identity and IDs](/docs/concepts/run-hierarchy/identity-and-ids).
