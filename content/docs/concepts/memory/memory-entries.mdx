---
title: Memory entries
description: The MemoryEntry row shape — `(tenant_id, project_id, scope_type, scope_id, namespace, key, value, retention, ttl, embedding?)` — and how entries are namespaced, written, read, and expired.
---

A **MemoryEntry** is a row in the [Memory Store](/docs/concepts/memory/memory-store): a typed key-value record rooted at a [MemoryScope](/docs/concepts/memory/memory-scopes), namespaced inside that scope, and addressable by key. Entries are the persistent backing for cross-Run, cross-Session context that survives beyond a single inference call.

The shape is LangGraph `Store`-compatible and backs Anthropic's memory tool's caller-provided persistence contract. Same row shape, two integration paths.

## Shape

| Field | Type | Purpose |
|---|---|---|
| `id` | ULID | Bare 26-char identity. |
| `tenant_id` | ULID | Owner. |
| `project_id` | ULID | Owner. |
| `scope_type` | enum | `ephemeral` / `session` / `user` / `tenant` / `global`. See [memory scopes](/docs/concepts/memory/memory-scopes). |
| `scope_id` | string | Concrete scope target — Session id, user id, tenant id, or deployment id. |
| `namespace` | string | Logical namespace within the scope (`preferences`, `goals`, `tools`, …). |
| `key` | string | Key within the namespace. |
| `value` | JSON | Structured value. |
| `retention` | enum | `with_call` / `with_session` / `with_compaction_lineage` / `with_tenant`. |
| `ttl_seconds` | int? | Optional explicit TTL. Overrides retention class when set. |
| `embedding` | float[]? | Optional vector for semantic search. |
| `created_at`, `updated_at` | timestamp | Audit. |
| `version` | int | Monotonic per `(scope, namespace, key)`. New writes increment. |

The composite key for addressing an entry is `(tenant_id, project_id, scope_type, scope_id, namespace, key)`. Two entries that share that key but differ on any field collide on the same logical row; new writes increment `version` and replace `value`.

## Namespacing

Within a scope, entries are organized by `namespace` then `key`:

```
scope: user / scope_id: user_42
├── namespace: preferences
│   ├── key: language        →  "en-US"
│   ├── key: tone            →  "concise"
│   └── key: timezone        →  "America/Los_Angeles"
├── namespace: goals
│   ├── key: 2026-q2         →  "ship a v2 release"
│   └── key: longterm        →  "..."
└── namespace: notes
    └── key: 01J8X9Y…        →  {...}
```

Namespaces are flat strings, not hierarchical. The Memory Store does not interpret `/` or `.` in a namespace; conventions like `notes/2026/q2` are flat namespace strings and `q2` is not a sub-namespace.

## Writing

```http
PUT /v1/ks/memory/entries
Authorization: Bearer ks_...
Content-Type: application/json

{
  "scope_type": "user",
  "scope_id": "user_42",
  "namespace": "preferences",
  "key": "language",
  "value": "en-US",
  "retention": "with_tenant"
}
```

```json
{
  "id": "01J8X9Y2Z3K4M5N6P7Q8R9ME11",
  "scope_type": "user",
  "scope_id": "user_42",
  "namespace": "preferences",
  "key": "language",
  "value": "en-US",
  "retention": "with_tenant",
  "version": 3,
  "created_at": "2026-04-30T12:00:00Z",
  "updated_at": "2026-04-30T12:00:00Z"
}
```

Writing the same `(scope, namespace, key)` is **last-write-wins** with version increment. The Memory Store does not support compare-and-swap or conditional writes.

Idempotency keys (supplied via the inference envelope or the explicit endpoint) make duplicate writes a no-op; without an idempotency key, the same value written twice produces two version increments.

## Reading

Single-entry read:

```http
GET /v1/ks/memory/entries?scope_type=user&scope_id=user_42&namespace=preferences&key=language
Authorization: Bearer ks_...
```

Namespace listing:

```http
GET /v1/ks/memory/entries?scope_type=user&scope_id=user_42&namespace=preferences
Authorization: Bearer ks_...
```

```json
{
  "entries": [
    { "key": "language", "value": "en-US", "version": 3 },
    { "key": "tone", "value": "concise", "version": 1 },
    { "key": "timezone", "value": "America/Los_Angeles", "version": 2 }
  ],
  "next_cursor": null
}
```

Cross-namespace listing requires a separate `namespace=*` form and is bounded by pagination.

## Selectors on the inference path

When a Run wants memory entries injected into its [ContextManifest](/docs/concepts/context/context-manifests), the runtime sends `memory_class_hints` on the `kovaserve` envelope:

```json
{
  "kovaserve": {
    "memory_class_hints": [
      {
        "selector": { "namespace": "preferences" },
        "memory_scope": "user",
        "retention": "with_call",
        "checkpoint_relevant": true
      }
    ]
  }
}
```

GCM v2 resolves the selector — typically `namespace` plus optionally `key` or a key prefix — and includes matching entries as `system_memory` [ContextSegments](/docs/concepts/context/context-segments) under canonical ordering. The runtime never inlines memory content directly; it asks for it by selector and lets GCM fetch and order.

## Retention and TTL

| `retention` | Lives until |
|---|---|
| `with_call` | The ModelCall that wrote it closes. Useful for one-shot scratch values. |
| `with_session` | The bound Session is `ARCHIVED` or `EXPIRED`. |
| `with_compaction_lineage` | Every CompactionRecord referencing this entry has been pruned. |
| `with_tenant` | Tenant retention policy decides. The default for `user` / `tenant` / `global` scopes. |

`ttl_seconds` overrides `retention` when set: an entry with `ttl_seconds: 3600` expires one hour after `updated_at`, regardless of retention class.

Expiry is **lazy** — reads check TTL and return `not_found` for expired entries; a background sweep deletes expired rows in bulk.

## Versioning and history

Entries are versioned, not append-only. The current version is what reads return; prior versions are not generally addressable through the public API. Audit-grade history is exposed via the `ks.memory.entry.written` event stream rather than versioned reads.

## Search

Optional `embedding` enables semantic search on entries. The dedicated search endpoint runs over a tenant- and scope-filtered subset:

```http
GET /v1/ks/memory/search?namespace=goals&query=ship+v2&scope_type=user&scope_id=user_42
```

Search requires the deployment to have a vector backend configured. When the backend is unavailable, the endpoint returns `503 Service Unavailable`; lookups by exact `(scope, namespace, key)` continue to work without the backend.

## Events

| Event | Emitted on |
|---|---|
| `ks.memory.entry.written` | Write or version increment. |
| `ks.memory.entry.read` | Read with `audit: true` flag. |
| `ks.memory.entry.expired` | Lazy or sweep expiry. |
| `ks.memory.entry.deleted` | Explicit delete. |

## What MemoryEntries are not

- **Not Session turn history.** Turn history lives on `SessionTurn` rows in the [Session](/docs/concepts/continuity/sessions) Registry. MemoryEntries are scope-rooted, not Session-rooted.
- **Not opaque blobs.** `value` is JSON. Large values belong in [ContextArtifacts](/docs/concepts/context/context-artifacts) referenced from a MemoryEntry pointer.
- **Not append-only.** Versions are last-write-wins. The event stream is the audit log.
- **Not free of scope.** Every entry has a non-null `scope_type`. There is no global default — `global` scope is the explicit scope, not the absence of one.
