A **Project** is the logical application boundary inside a [Tenant](/docs/concepts/tenancy/tenants). It is the level at which workloads are organized, cache reuse is scoped (by default), and policy overlays land for a coherent product line. Every [Session](/docs/concepts/continuity/sessions), [Run](/docs/concepts/run-hierarchy/runs), and [ContextArtifact](/docs/concepts/context/context-artifacts) belongs to exactly one Project.

A typical mapping is one Project per application or product surface вЂ” `chatbot-prod`, `coding-agent`, `contract-review`. The intent is that workloads inside a Project share a runtime model вЂ” the same agent code, the same tool catalog version, the same compaction policy вЂ” even when they run across multiple [Environments](/docs/concepts/tenancy/environments).

## What a Project owns

| Field | Purpose |
|---|---|
| `id` | Bare 26-char Crockford ULID. |
| `tenant_id` | Parent Tenant. |
| `name` | Human-readable identifier, unique within the Tenant. |
| `status` | `ACTIVE` or `ARCHIVED`. Archived Projects reject new admission but remain queryable. |
| `policy_scope_id` | Project-level [PolicyScope](/docs/concepts/policy/policy-scopes) override. Narrows against the Tenant default. |
| `default_context_share_scope` | Cache and manifest reuse boundary applied to Runs in this Project: `none` / `project` / `tenant`. Default `project`. |
| `default_compaction_policy` | Compaction strategy hint for Runs that do not declare their own вЂ” see [compaction records](/docs/concepts/context/compaction-records). |
| `created_at` | Timestamp. |

A Project name is unique within its Tenant. Creating a Project with a name already in use returns `409 Conflict` with `error.code = "project_name_conflict"`.

## Cache and manifest reuse

The Project is the **default cache namespace**. The [Cache-Aware Router](/docs/concepts/routing/manifest-aware-routing) only considers replicas warmed by the same Project unless the resolved [PolicyScope's](/docs/concepts/policy/policy-scopes#context_share_scope) `context_share_scope` widens to `tenant` (Tenant-wide) or narrows to `none` (per-Run private namespace).

Practical effect: two Runs in the same Project that share a system prompt, tool catalog, and history-prefix tier hit the same cache. Two Runs in different Projects within the same Tenant do not, by default вЂ” even if their content matches byte-for-byte.

## Inheritance and overlay

| Resolved value | Resolution rule |
|---|---|
| PolicyScope | Project's `policy_scope_id` overlays the Tenant default. Narrowing only. |
| Model ACL | Intersection of Tenant ACL and Project ACL. A Project cannot reach a model the Tenant is not contracted for. |
| Rate limits | Project limits cap traffic from this Project. The Tenant limit caps the sum of all Projects underneath. |
| Data boundary | Most-restrictive of Tenant default and Project setting. |
| Compaction policy | Project default; Run-level policy hints override per-Run. |

Resolution is performed once at admission time and frozen on the Run row. See [PolicyScope](/docs/concepts/policy/policy-scopes#resolution).

## Aggregation

Project-level totals are sums over `usage_records` rows tagged with the Project's `project_id`:

| Surface | Endpoint |
|---|---|
| Usage and cost | `GET /v1/ks/metering/usage?scope=project&project_id=...` |
| Per-day breakdown | `GET /v1/ks/metering/usage?scope=project&project_id=...&group_by=day` |
| Per-Run breakdown | `GET /v1/ks/metering/usage?scope=project&project_id=...&group_by=run` |
| JSONL export | `GET /v1/ks/metering/export?scope=project&project_id=...` |

`scope=project` with an unknown `project_id` returns `200` with zero counts, not `404` вЂ” see [ADR-023 error semantics](/docs/concepts/cost/cost-model#aggregation).

## Archival

A Project can be moved to `ARCHIVED`. The effect is structural:

| Surface | Behavior under ARCHIVED |
|---|---|
| New Run admission | Reject with `403 Forbidden`, `error.code = "project_archived"`. |
| In-flight Runs | Continue to completion. Steps already admitted are not interrupted. |
| Read-only endpoints | Continue to serve indefinitely. |
| New Keys | Cannot be issued against an Archived Project. |
| Existing Keys | Continue to authenticate but every admission attempt rejects. |

Archival is reversible: an ARCHIVED Project can be moved back to ACTIVE. Existing data вЂ” Sessions, Runs, ContextArtifacts вЂ” is not touched.

## What a Project is not

- **Not the tenancy root.** The Tenant is the tenancy root; the Project is one level beneath.
- **Not a deployment.** A Project's Runs may execute against multiple deployments (per Environment) вЂ” see [environments](/docs/concepts/tenancy/environments).
- **Not a cache key.** The cache namespace is computed from `(tenant_id, project_id, context_share_scope)` plus the manifest's `stable_context_id`. The Project ID alone does not key cache lookups; it scopes them.
- **Not deletable in-place.** Projects are archived, not deleted. Hard deletion of historical data follows the data-retention contract on the Tenant, not a Project-level operation.
