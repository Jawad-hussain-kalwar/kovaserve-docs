title: Context budgets
description: The ContextBudget entity — per-Run admission control over active context size, evaluated against every assembled ContextManifest before GPU dispatch.
---

A **ContextBudget** is a per-[Run](/docs/concepts/runs) admission control object that bounds active context size. It is evaluated against every assembled [ContextManifest](/docs/concepts/context-manifests) **before** GPU dispatch and produces explicit decisions: `ALLOW`, `WARN`, `OFFLOAD_REQUIRED`, `COMPACT_REQUIRED`, `DEGRADE`, or `HARD_STOP`.

ContextBudget is KovaServe-novel — no vendor exposes context-token admission as a first-class API construct. It exists because a Run-level [Budget](/docs/concepts/budgets) governing cost or step count is not enough to keep agent runs operationally bounded: the failure mode that matters most is the *active prompt* growing past what the model can handle or what the budget can afford.

## Dimensions

| Dimension | Limit field | Source on the manifest |
|---|---|---|
| Active tokens | `active_token_limit` | `token_counts.active_tokens` |
| Stable tokens | `stable_token_limit` | `token_counts.stable_tokens` |
| Volatile tokens | `volatile_token_limit` | `token_counts.volatile_tokens` |
| Tool result tokens | `tool_result_token_limit` | sum over `tool_output` segments |
| Artifact bytes | `artifact_byte_limit` | `byte_counts.artifact_bytes` |
| Compaction trigger | `compaction_required_threshold` | active token consumption fraction |
| Offload trigger | `offload_required_threshold` | per-segment inline byte threshold |

Limits left unset are unbounded for that dimension. A typical ContextBudget pins `active_token_limit` and the two trigger thresholds; finer dimensions are configured for governed workloads.

## Decisions

The Context Admission Service emits exactly one decision per assembled manifest:

| Decision | Effect | Transient? |
|---|---|---|
| `ALLOW` | Manifest proceeds to substrate dispatch unchanged. | terminal-allow |
| `WARN` | Proceeds; emits `ks.context.budget.warning`. | informational |
| `OFFLOAD_REQUIRED` | Oversized inline segment is rewritten to an artifact reference; manifest re-admitted. | transient advisory |
| `COMPACT_REQUIRED` | Runtime (or managed compactor) compacts; manifest re-admitted. | transient advisory |
| `DEGRADE` | `max_tokens` reduced and/or optional segments dropped; manifest re-admitted. | transient advisory |
| `HARD_STOP` | Manifest rejected before GPU dispatch. ModelCall is not created. | terminal |

The four advisory decisions (`OFFLOAD_REQUIRED`, `COMPACT_REQUIRED`, `DEGRADE`, plus `WARN` informationally) are **transient** — the runtime applies the indicated transformation and re-admits. Only `HARD_STOP` is terminal.

## State machine

```
ACTIVE ──▶ WARN ──▶ ACTIVE                           (transient warning)
ACTIVE ──▶ OFFLOAD_REQUIRED ──▶ ACTIVE               (offload applied, re-admitted)
ACTIVE ──▶ COMPACT_REQUIRED ──▶ ACTIVE               (compaction applied, re-admitted)
ACTIVE ──▶ DEGRADE ──▶ ACTIVE                        (degraded shape, re-admitted)
ACTIVE ──▶ HARD_STOP ──▶ CLOSED                      (terminal, request rejected)
```

The non-`ACTIVE` non-terminal states are FSM annotations of the action being taken. On the wire, the same identifiers double as the `ContextBudgetDecision` return type from the admission service. See the [context-budget state machine](/docs/concepts/context-budget-state-machine) for the full graph.

## Run-level vs context-level

| | [Budget](/docs/concepts/budgets) | ContextBudget |
|---|---|---|
| Governs | Cost / tokens / steps / time / tool calls | Active context size |
| Scope | Whole Run | Per-ModelCall manifest |
| Evaluation order | After ContextBudget | First — before any cost work |
| Re-admission | Not applicable; Budget is informational on subsequent Steps | Common — most decisions are advisory |
| Storage | Single row per Run | Single row per Run; admission decisions recorded on each manifest |

Both must `ALLOW` for a ModelCall to dispatch.

## Inheritance

| Operation | ContextBudget behavior |
|---|---|
| [Branch](/docs/concepts/branches) | Inherited from parent (overridable downward). |
| [Sub-run](/docs/concepts/sub-runs) | Allocated explicitly by parent; child runs under its own ContextBudget. |
| [Resume](/docs/concepts/resume-flow) | Inherited unchanged. |

Limits cannot be raised on inheritance.

## Events

| Event | Emitted on |
|---|---|
| `ks.context.budget.warning` | `WARN` decision. |
| `ks.context.budget.offload_required` | `OFFLOAD_REQUIRED` decision. |
| `ks.context.budget.compact_required` | `COMPACT_REQUIRED` decision. |
| `ks.context.budget.degraded` | `DEGRADE` decision. |
| `ks.context.budget.hard_stop` | `HARD_STOP` decision. |

## What ContextBudgets are not

- **Not a substitute for [Budgets](/docs/concepts/budgets).** They govern different things; both must `ALLOW` for dispatch.
- **Not a runtime concern.** The runtime supplies content; admission is owned by the Context Admission Service. Runtimes react to advisory decisions by transforming their input and retrying.
- **Not silent.** Every decision is recorded on the manifest and emitted as an event.
- **Not retroactive.** A `HARD_STOP` rejects the request before dispatch; it does not cancel in-flight ModelCalls.
