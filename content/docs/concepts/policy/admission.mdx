**Admission** is the pre-dispatch evaluation that decides whether an assembled [ContextManifest](/docs/concepts/context/context-manifests) may proceed to substrate execution, and if so, in what shape. It is owned by the **Context Admission Service** and runs on every ModelCall. The service consumes the assembled manifest, evaluates it against the Run's [ContextBudget](/docs/concepts/policy/context-budgets), applies any required transformations, and returns the admitted shape.

Admission is the single point at which "can this run, and how" is decided. After admission, dispatch is unconditional â€” the substrate sees only manifests that admission has cleared.

## Where admission sits in the hot path

```
1. Auth / scope resolution
2. Run / Session resolution
3. Context inputs gathered (history, artifact previews)
4. ContextManifest assembled
5. â–¶â–¶ ContextBudget admission â—€â—€          (this page)
6. Run-level Budget pre-check on the admitted manifest
7. Step is created, bound to the manifest
8. Manifest-aware routing
9. Substrate execution
```

Admission runs *after* manifest assembly so it can score real token counts and real artifact byte counts. It runs *before* the run-level [Budget](/docs/concepts/policy/budgets) pre-check so cost is computed against the admitted shape, not the raw assembly.

## Decisions

The service emits exactly one decision per evaluation:

| Decision | Effect |
|---|---|
| `ALLOW` | Proceed unchanged. |
| `WARN` | Proceed; emit `ks.context.budget.warning`. Informational. |
| `OFFLOAD_REQUIRED` | Apply offload (oversized inline segment â†’ artifact reference); re-admit. |
| `COMPACT_REQUIRED` | Runtime or managed compactor compacts; re-admit. |
| `DEGRADE` | Reduce `max_tokens` and/or drop optional segments; re-admit. |
| `HARD_STOP` | Reject before dispatch. ModelCall is not created. |

`OFFLOAD_REQUIRED`, `COMPACT_REQUIRED`, and `DEGRADE` are **transient advisories** â€” the indicated transformation is applied and the manifest is re-admitted. Re-admission either reaches `ALLOW`/`WARN` (proceed) or produces another advisory. The service caps the re-admission chain (typical default: three rounds) to bound work; an unresolvable chain falls through to `HARD_STOP`.

## The evaluation

For each dimension on the [ContextBudget](/docs/concepts/policy/context-budgets), admission computes the consumed ratio and applies threshold rules:

```
ratio = manifest.token_counts.<dim> / context_budget.<dim>_limit

ratio < warn_threshold       â†’ ALLOW
warn â‰¤ ratio < hard           â†’ WARN  (or transformation if a trigger fires)
ratio â‰Ą hard                  â†’ HARD_STOP

if any segment.byte_count > offload_required_threshold
                              â†’ OFFLOAD_REQUIRED

if active_token_ratio â‰Ą compaction_required_threshold
                              â†’ COMPACT_REQUIRED

if degrade_eligible AND no transformation viable AND no HARD_STOP
                              â†’ DEGRADE
```

Multiple conditions can be satisfied; precedence is: `HARD_STOP` > `OFFLOAD_REQUIRED` > `COMPACT_REQUIRED` > `DEGRADE` > `WARN` > `ALLOW`. The service emits the highest-precedence decision and applies the matching transformation.

## Transformations

| Decision | Transformation |
|---|---|
| `OFFLOAD_REQUIRED` | Each segment exceeding `offload_required_threshold` is rewritten: full payload â†’ [ContextArtifact](/docs/concepts/context/context-artifacts), segment row keeps a preview + `artifact_ref_id`. Manifest re-admits with reduced `active_tokens` and increased `artifact_bytes`. |
| `COMPACT_REQUIRED` | The runtime is signalled to compact (or the managed compactor runs). A [CompactionRecord](/docs/concepts/context/compaction-records) is registered. Compacted segments replace their sources in the next assembled manifest. |
| `DEGRADE` | `max_tokens` is reduced according to `context_budget.degrade_policy`; optional segments (those with `degrade_droppable: true`) are dropped. Manifest re-admits. |

Each transformation is recorded on the manifest's `admission.transformations[]` field so the path from raw assembly to admitted shape is reconstructable.

## The admitted manifest

After admission completes (with `ALLOW` or `WARN`), a final manifest row is written with:

- The post-transformation segment list and digests.
- Final `token_counts` and `byte_counts`.
- `admission.decision = ALLOW` (or `WARN`).
- `admission.transformations[]` listing every transformation applied across re-admission rounds.
- `admission.timestamp`.

The original pre-transformation manifest, if any, remains addressable in the Context Manifest Registry â€” admission writes a new manifest record on each round, never mutates an earlier one.

## Preflight admission

Runtimes can request a preflight admission decision without dispatching a ModelCall:

```http
POST /v1/ks/context/admit
Authorization: Bearer ks_...
Content-Type: application/json

{
  "run_id": "01J8X9Y2Z3K4M5N6P7Q8R9S0T1",
  "context_manifest": {
    "segments": [...]
  }
}
```

```json
{
  "decision": "OFFLOAD_REQUIRED",
  "transformations": [
    {
      "type": "artifact_offload",
      "segment_ids": ["01J8X9Y2Z3K4M5N6P7Q8R9SG07"]
    }
  ],
  "admitted_manifest": {
    "token_counts": { "active_tokens": 18420, "offloaded_tokens": 9412 }
  }
}
```

Preflight is useful for runtimes that want to plan transformations in advance (e.g. compact ahead of admission failure) rather than react to the inline decision on `/v1/chat/completions`.

## Events

| Event | Emitted on |
|---|---|
| `ks.context.manifest.admitted` | Final admission decision (`ALLOW` or `WARN`). |
| `ks.context.budget.offload_required` | `OFFLOAD_REQUIRED` decision. |
| `ks.context.budget.compact_required` | `COMPACT_REQUIRED` decision. |
| `ks.context.budget.degraded` | `DEGRADE` decision. |
| `ks.context.budget.hard_stop` | `HARD_STOP` decision. |
| `ks.context.budget.warning` | `WARN` decision. |

## Failure modes

| Failure | Outcome |
|---|---|
| Re-admission cap exceeded | Last decision falls through to `HARD_STOP` with `error.code = "context_budget_exceeded"`, HTTP `429`. |
| Transformation fails (artifact write failure, compactor unreachable) | `HARD_STOP` with `error.code = "admission_transformation_failed"`, HTTP `429`. |
| Manifest assembly produced an invalid shape | Rejected with `error.code = "manifest_invalid"`, HTTP `400 Bad Request`. ModelCall is not created. |

The canonical `HARD_STOP` rejection is HTTP `429 Too Many Requests` with `error.code = "context_budget_exceeded"` and an explanatory body â€” a client-side rate signal, not a server error. Every response carries an `X-KS-Context-Budget-State` header so runtimes can observe the post-evaluation state without re-reading the budget.
