title: Context engineering events

The **context engineering events** stream covers the lifecycle of every context-engineering primitive: [manifests](/docs/concepts/context/context-manifests), [segments](/docs/concepts/context/context-segments), [artifacts](/docs/concepts/context/context-artifacts), [compaction records](/docs/concepts/context/compaction-records), and [ContextBudget](/docs/concepts/policy/context-budgets) admission decisions.

Subjects: `ks.context.*`. Persisted in the `execution_events` JetStream stream. Consumed via [`GET /v1/ks/events/context-engineering`](/docs/api/events/context-engineering).

## Event types

### Manifest events

| Event | Trigger |
|---|---|
| `ks.context.manifest.created` | A new ContextManifest row is inserted. |
| `ks.context.manifest.admitted` | Admission settles with `ALLOW` or `WARN`. The admitted manifest is the one that proceeds to dispatch. |

### Segment events

| Event | Trigger |
|---|---|
| `ks.context.segment.created` | A ContextSegment row is inserted. |
| `ks.context.segment.evicted` | A segment is removed under retention or compaction. |

### Artifact events

| Event | Trigger |
|---|---|
| `ks.context.artifact.created` | A ContextArtifact row is inserted (object-storage upload completed). |
| `ks.context.artifact.offloaded` | An inline segment was rewritten as an artifact reference under `OFFLOAD_REQUIRED`. |

### Compaction events

| Event | Trigger |
|---|---|
| `ks.context.compaction.started` | A CompactionRecord is opened. |
| `ks.context.compaction.completed` | Compaction finished; output segments are usable. |

### ContextBudget events

| Event | Trigger |
|---|---|
| `ks.context.budget.warning` | Admission produced `WARN`. |
| `ks.context.budget.offload_required` | Admission produced `OFFLOAD_REQUIRED`. |
| `ks.context.budget.compact_required` | Admission produced `COMPACT_REQUIRED`. |
| `ks.context.budget.degraded` | Admission produced `DEGRADE`. |
| `ks.context.budget.hard_stop` | Admission produced `HARD_STOP`. |

### `ks.context.budget.*`

All ContextBudget events share a payload shape:

```json
{
  "context_budget_id": "01J8X9Y2Z3K4M5N6P7Q8R9CB01",
  "decision": "OFFLOAD_REQUIRED",
  "dimension": "active_tokens",
  "limit": 16384,
  "consumed": 18420,
  "consumed_pct": 112.43,
  "transformation_planned": "artifact_offload"
}
```

`decision` matches the event type's suffix. `transformation_planned` is the action the [admission](/docs/concepts/policy/admission) service is about to apply.

## Sequence around a single ModelCall

```
ks.context.segment.created      (per inbound segment)
ks.context.manifest.created     (GCM v2 produces the initial manifest)
[optional] ks.context.budget.offload_required
ks.context.artifact.offloaded   (one per offloaded segment)
[optional] ks.context.budget.compact_required
ks.context.compaction.started
ks.context.compaction.completed
ks.context.manifest.admitted    (final admitted shape)
```

The sequence may include zero, one, or multiple admission round-trips depending on transformations applied. The `ks.context.manifest.admitted` event always lands last for the call, with the final decision (`ALLOW` or `WARN`).

## Identity fields

The envelope's `context_manifest_id` is populated on every event tied to a specific manifest. `run_id`, `step_id`, and `model_call_id` are populated for events fired in the per-call hot path (manifest assembly, admission, compaction triggered during the call). Standalone artifact events may have `run_id` and `model_call_id` null when the write is not call-bound.

## Common consumers

- **Context timeline UI.** Per-Run timeline of segment composition, manifest evolution, and admission decisions.
- **Compaction lineage view.** Walks `ks.context.compaction.completed` events by `source_manifest_id` to draw the lineage tree.
- **ContextBudget dashboards.** Aggregate `ks.context.budget.*` decisions per Project to surface budget tuning needs.
- **Cache reuse audit.** Joins `ks.context.manifest.admitted.stable_context_id` against [state events](/docs/concepts/events/state) `kv.state.accessed.stable_context_id` to verify reuse predictions.
