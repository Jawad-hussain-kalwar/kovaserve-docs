KovaServe is a control plane for long-running AI execution. Its public surface вЂ” REST API, SDK, CLI, console вЂ” is a thin shell over a small set of canonical entities that mean the same thing in managed cloud and self-hosted deployments.

This section defines those entities. Everything in the API and SDK references resolves back to terms introduced here.

## The canonical entities

| Entity | What it is |
|---|---|
| **Run** | The top-level durable execution objective. The unit of value. May have a `parent_run_id` (sub-run lineage). |
| **Session** | A continuity container across turns or related runs. Tracks the current `ContextManifest`. |
| **Step** | A typed unit of progression inside a Run. See [step types](/docs/concepts/runs-steps-model-calls#step-types). |
| **ModelCall** | One substrate-bound model invocation belonging to a Step. Carries a `context_manifest_id`. |
| **ContextManifest** | A durable per-call record of the context shape вЂ” segments, token counts, stable identity. The audit anchor for budgets, routing, checkpoints, and resume. |
| **Checkpoint** | A resumable snapshot binding logical context first; KV cache export is an optimization layered on top. |
| **StateHandle** | A typed reference to reusable execution state: `kv_prefix`, `context_manifest`, `checkpoint`, `artifact_bundle`, `memory_entry`, or `runtime_state`. |
| **Budget** | A run-level execution budget controlling cost, tokens, steps, time, or tool count. |
| **ContextBudget** | A per-run admission control object for active context size, evaluated before GPU dispatch. |
| **PolicyScope** | Normalized governance binding across tenant / project / environment / deployment / runtime. |
| **MemoryScope** | The retention class for context and state: `ephemeral`, `session`, `user`, `tenant`, or `global`. |

## Three rules that govern the model

1. **The Run is the unit of value.** Not a token, not a request. A Run completes when the work it represents is done вЂ” possibly after many Steps, ModelCalls, pauses, resumes, and checkpoints.
2. **Continuity is first-class.** Sessions, ContextManifests, Checkpoints, and StateHandles exist so that long-running work survives node restarts, failures, and movement across infrastructure without losing logical state.
3. **Admission happens before execution.** PolicyScope, Budget, and ContextBudget evaluate against the assembled ContextManifest *before* GPU dispatch. Decisions are explicit (`ALLOW`, `WARN`, `OFFLOAD_REQUIRED`, `COMPACT_REQUIRED`, `DEGRADE`, `HARD_STOP`) and traceable.
