---
title: List Environments
description: GET /v1/ks/environments — page through Environments with cursor pagination, filtered by Project, tier, status, or name prefix.
---

`GET /v1/ks/environments` lists [Environments](/docs/concepts/tenancy/environments) under the authenticated Tenant, newest first. The response uses cursor pagination — the same shape as every other list endpoint in the API. Filters narrow the result to a single Project, tier, status, or name prefix.

The Tenant filter is implicit. Environments in other Tenants never appear, regardless of any query parameter combination.

## Endpoint

```
GET /v1/ks/environments
```

| Property | Value |
|---|---|
| Auth | `Authorization: Bearer ks_...` — see [Authentication](/docs/api/using-the-api/authentication). |
| Pagination | Cursor — see [Pagination](/docs/api/using-the-api/pagination). |
| Errors | Native envelope — see [Errors](/docs/api/using-the-api/errors). |

## Query parameters

| Parameter | Type | Default | Meaning |
|---|---|---|---|
| `limit` | integer | `100` | Maximum Environments to return. Capped at `200`. Values outside `1..200` return `400 invalid_request`. |
| `after` | string \| null | `null` | Cursor — return Environments strictly after it in descending creation order. |
| `before` | string \| null | `null` | Cursor — return Environments strictly before it. Mutually exclusive with `after`. |
| `project_id` | string \| null | `null` | Restrict to one Project. |
| `tier` | string \| null | `null` | Restrict to one of `prod` / `staging` / `dev`. |
| `status` | string \| null | `null` | Restrict to one of `ACTIVE` / `ARCHIVED`. |
| `name_prefix` | string \| null | `null` | Restrict to Environments whose `name` starts with the prefix. Case-sensitive. |
| `created_after` | string \| null | `null` | ISO-8601 timestamp — Environments with `created_at >= created_after`. |
| `created_before` | string \| null | `null` | ISO-8601 timestamp — Environments with `created_at < created_before`. |

## Status codes

| Status | When |
|---|---|
| `200 OK` | Returns a page of Environments. Empty `data` is valid. |
| `400 Bad Request` | Both `before` and `after` supplied, `limit` outside `1..200`, an unrecognized `tier` or `status`, or a malformed timestamp. Code: `invalid_request`. |
| `401 Unauthorized` | Missing or invalid bearer token. |
| `404 Not Found` | A scoping ID (`project_id`) does not resolve in the caller's Tenant. Code: `project_not_found`. |
| `429 Too Many Requests` | Rate limit exceeded. Code: `rate_limited`. |

## Examples

```bash
# Every Environment under one Project
curl "https://api.kovaserve.com/v1/ks/environments?project_id=01J8X9Y2Z3K4M5N6P7Q8R9PRJ1" \
  -H "Authorization: Bearer ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b"

# All production-tier Environments across the Tenant
curl "https://api.kovaserve.com/v1/ks/environments?tier=prod" \
  -H "Authorization: Bearer ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b"

# Archived Environments awaiting cleanup
curl "https://api.kovaserve.com/v1/ks/environments?status=ARCHIVED" \
  -H "Authorization: Bearer ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b"
```

```python
from kovaserve import KovaClient

kova = KovaClient(api_key="ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b")

# Every production-tier Environment, page-by-page
cursor = None
while True:
    page = kova.environments.list(tier="prod", limit=100, after=cursor)
    for e in page.data:
        print(f"{e.name:14} {e.project_id}  status={e.status}")
    if not page.has_more:
        break
    cursor = page.next_cursor

# Built-in iterator helper — auto-paginates
for e in kova.environments.iter(project_id="01J8X9Y2Z3K4M5N6P7Q8R9PRJ1"):
    print(f"{e.tier:8} {e.name}")
```
