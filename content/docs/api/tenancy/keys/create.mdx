---
title: Create a Key
description: POST /v1/ks/keys — issue a Bearer key bound to one (Tenant, Project, Environment) triple. The raw secret is returned once and never again.
---

`POST /v1/ks/keys` issues a Bearer [Key](/docs/concepts/tenancy/keys) bound to exactly one `(tenant_id, project_id, environment_id)` triple. The triple is **immutable for the Key's lifetime** — promotion across Projects or Environments is done by issuing a new Key and revoking the old one.

The response is the **only time the raw secret is returned**. KovaServe stores the SHA-256 hash of the secret; the raw value cannot be recovered. Lost Keys must be [rotated](/docs/api/tenancy/keys/rotate), not retrieved. Clients must capture the `secret` field at issuance.

## Endpoint

```
POST /v1/ks/keys
```

| Property | Value |
|---|---|
| Auth | `Authorization: Bearer ks_...` — must be an admin-scoped Key. See [Authentication](/docs/api/using-the-api/authentication). |
| Content type | `application/json`. |
| Idempotent | Yes, when `Idempotency-Key` is supplied — see [Idempotency](/docs/api/using-the-api/idempotency). |
| Errors | Native envelope — see [Errors](/docs/api/using-the-api/errors). |

## Request body

| Field | Type | Required | Meaning |
|---|---|---|---|
| `project_id` | string | yes | The Project the Key authenticates against. Must be visible to the API key's Tenant and not in `ARCHIVED`. |
| `environment_id` | string | yes | The Environment the Key authenticates against. Must belong to the supplied Project and not in `ARCHIVED`. |
| `name` | string | yes | Human-readable label. 1–64 characters. Surfaces in dashboards and audit logs. |
| `mode` | string | no | `live` or `test`. Defaults to the Environment's tier — `prod` / `staging` map to `live`; `dev` maps to `test`. A `test` Key rejects against `prod` deployments with `error.code = "key_mode_environment_mismatch"`. |
| `expires_at` | string \| null | no | ISO-8601 timestamp. The Key transitions to `EXPIRED` at this instant and rejects with `error.code = "key_expired"`. `null` = never expires. |

The `tenant_id` is implicit — taken from the authenticated key. Unknown top-level fields return `400 invalid_request` with `details.field_errors` listing the offending keys.

### Example

```json
{
  "project_id": "01J8X9Y2Z3K4M5N6P7Q8R9PRJ1",
  "environment_id": "01J8X9Y2Z3K4M5N6P7Q8R9ENV1",
  "name": "ci-bot",
  "mode": "live",
  "expires_at": "2027-04-30T00:00:00Z"
}
```

## Response — `201 Created`

The raw secret appears in the response **once**. Subsequent reads of the Key omit the `secret` field.

```json
{
  "id": "01J8X9Y2Z3K4M5N6P7Q8R9KEY1",
  "tenant_id": "01J8X9Y2Z3K4M5N6P7Q8R9ORG1",
  "project_id": "01J8X9Y2Z3K4M5N6P7Q8R9PRJ1",
  "environment_id": "01J8X9Y2Z3K4M5N6P7Q8R9ENV1",
  "name": "ci-bot",
  "mode": "live",
  "status": "ACTIVE",
  "last4": "e1f2",
  "secret": "ks_live_3f2a8b7e6d4c1f0925a1b6c3d4e5f6a7b8c9d0e1f2",
  "created_at": "2026-05-05T12:00:00Z",
  "expires_at": "2027-04-30T00:00:00Z",
  "last_used_at": null
}
```

| Field | Type | Meaning |
|---|---|---|
| `id` | string | Key ID — bare 26-character Crockford ULID. The `key_id` referenced in events, audit logs, and PolicyScope resolution. |
| `tenant_id` / `project_id` / `environment_id` | string | The bound triple. Immutable. |
| `name` | string | Echo of the request `name`. Updatable via `PATCH /v1/ks/keys/{id}`. |
| `mode` | string | `live` or `test`. |
| `status` | string | `ACTIVE`, `REVOKED`, or `EXPIRED`. New Keys return in `ACTIVE`. |
| `last4` | string | Last four characters of the secret. Used for dashboard display. |
| `secret` | string | **Raw secret. Returned only on creation and rotation.** Stored client-side; KovaServe holds only the SHA-256 hash. |
| `created_at` | string | ISO-8601 timestamp with `Z` suffix. |
| `expires_at` | string \| null | Echo of the request field. |
| `last_used_at` | string \| null | Updated on every successful authentication. `null` until first use. |

### Headers

```http
HTTP/1.1 201 Created
Content-Type: application/json
X-Request-Id: 01J8X9Y2Z3K4M5N6P7Q8R9S0XX
X-KS-Key-Id:  01J8X9Y2Z3K4M5N6P7Q8R9KEY1
```

## Status codes

| Status | When |
|---|---|
| `201 Created` | The Key was issued and is immediately usable for authentication. |
| `400 Bad Request` | Malformed JSON, unknown fields, malformed IDs, unknown `mode`, or an `expires_at` in the past. Code: `invalid_request`. |
| `401 Unauthorized` | Missing, malformed, or unresolved API key. Codes: `missing_token`, `invalid_token`, `token_revoked`. |
| `403 Forbidden` | The authenticated Tenant is `SUSPENDED`, the target Project is `ARCHIVED`, or the target Environment is `ARCHIVED`. Codes: `tenant_suspended`, `project_archived`, `environment_archived`. |
| `404 Not Found` | The supplied `project_id` or `environment_id` does not resolve in the caller's Tenant, or the Environment does not belong to the supplied Project. Codes: `project_not_found`, `environment_not_found`. |
| `409 Conflict` | The supplied `Idempotency-Key` was reused with a different request body. Code: `idempotency_conflict`. |
| `429 Too Many Requests` | Rate limit exceeded. Code: `rate_limited`. |

## Examples

```bash
# Production Key for a CI bot
curl -X POST https://api.kovaserve.com/v1/ks/keys \
  -H "Authorization: Bearer ks_admin_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "01J8X9Y2Z3K4M5N6P7Q8R9PRJ1",
    "environment_id": "01J8X9Y2Z3K4M5N6P7Q8R9ENV1",
    "name": "ci-bot",
    "mode": "live",
    "expires_at": "2027-04-30T00:00:00Z"
  }'

# Short-lived development Key, with idempotency
curl -X POST https://api.kovaserve.com/v1/ks/keys \
  -H "Authorization: Bearer ks_admin_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b" \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: 6c1f8a30-9b22-4f0e-9d6c-12c4f0a1b2c3" \
  -d '{
    "project_id": "01J8X9Y2Z3K4M5N6P7Q8R9PRJ1",
    "environment_id": "01J8X9Y2Z3K4M5N6P7Q8R9ENV0",
    "name": "alice-laptop",
    "mode": "test",
    "expires_at": "2026-06-05T00:00:00Z"
  }'
```

```python
from kovaserve import KovaClient
import os

admin = KovaClient(api_key=os.environ["KOVASERVE_ADMIN_KEY"])

issued = admin.keys.create(
    project_id="01J8X9Y2Z3K4M5N6P7Q8R9PRJ1",
    environment_id="01J8X9Y2Z3K4M5N6P7Q8R9ENV1",
    name="ci-bot",
    mode="live",
    expires_at="2027-04-30T00:00:00Z",
)

# Capture the raw secret immediately — it is returned only here.
write_secret_to_vault(issued.id, issued.secret)

print(f"issued {issued.id} (last4={issued.last4})")
```
