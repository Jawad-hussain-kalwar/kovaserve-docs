---
title: List Keys
description: GET /v1/ks/keys — page through Keys with cursor pagination, filtered by Project, Environment, status, mode, or name prefix.
---

`GET /v1/ks/keys` lists [Keys](/docs/concepts/tenancy/keys) under the authenticated Tenant, newest first. The response uses cursor pagination — the same shape as every other list endpoint in the API. Filters narrow the result to a single Project, Environment, status, mode, or name prefix.

The Tenant filter is implicit. Keys in other Tenants never appear, regardless of any query parameter combination. Raw secrets are never included — list responses carry the same metadata shape as [`GET /v1/ks/keys/{id}`](/docs/api/tenancy/keys/get).

## Endpoint

```
GET /v1/ks/keys
```

| Property | Value |
|---|---|
| Auth | `Authorization: Bearer ks_...` — see [Authentication](/docs/api/using-the-api/authentication). |
| Pagination | Cursor — see [Pagination](/docs/api/using-the-api/pagination). |
| Errors | Native envelope — see [Errors](/docs/api/using-the-api/errors). |

## Query parameters

| Parameter | Type | Default | Meaning |
|---|---|---|---|
| `limit` | integer | `100` | Maximum Keys to return. Capped at `200`. Values outside `1..200` return `400 invalid_request`. |
| `after` | string \| null | `null` | Cursor — return Keys strictly after it in descending creation order. |
| `before` | string \| null | `null` | Cursor — return Keys strictly before it. Mutually exclusive with `after`. |
| `project_id` | string \| null | `null` | Restrict to one Project. |
| `environment_id` | string \| null | `null` | Restrict to one Environment. |
| `status` | string \| null | `null` | Restrict to one of `ACTIVE` / `REVOKED` / `EXPIRED`. |
| `mode` | string \| null | `null` | Restrict to one of `live` / `test`. |
| `name_prefix` | string \| null | `null` | Restrict to Keys whose `name` starts with the prefix. Case-sensitive. |
| `created_after` | string \| null | `null` | ISO-8601 timestamp — Keys with `created_at >= created_after`. |
| `created_before` | string \| null | `null` | ISO-8601 timestamp — Keys with `created_at < created_before`. |

## Status codes

| Status | When |
|---|---|
| `200 OK` | Returns a page of Keys. Empty `data` is valid. |
| `400 Bad Request` | Both `before` and `after` supplied, `limit` outside `1..200`, an unrecognized `status` or `mode`, or a malformed timestamp. Code: `invalid_request`. |
| `401 Unauthorized` | Missing or invalid bearer token. |
| `404 Not Found` | A scoping ID (`project_id`, `environment_id`) does not resolve in the caller's Tenant. Codes: `project_not_found`, `environment_not_found`. |
| `429 Too Many Requests` | Rate limit exceeded. Code: `rate_limited`. |

## Examples

```bash
# All ACTIVE production Keys
curl "https://api.kovaserve.com/v1/ks/keys?status=ACTIVE&mode=live" \
  -H "Authorization: Bearer ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b"

# Every Key bound to one Environment
curl "https://api.kovaserve.com/v1/ks/keys?environment_id=01J8X9Y2Z3K4M5N6P7Q8R9ENV1" \
  -H "Authorization: Bearer ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b"

# Recently revoked Keys, for audit review
curl "https://api.kovaserve.com/v1/ks/keys?status=REVOKED&created_after=2026-04-01T00:00:00Z" \
  -H "Authorization: Bearer ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b"
```

```python
from kovaserve import KovaClient

kova = KovaClient(api_key="ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b")

# Surface dormant Keys — ACTIVE but not used in the trailing 7 days
for k in kova.keys.iter(status="ACTIVE"):
    if k.request_count_7d == 0:
        print(f"dormant: {k.id}  {k.name}  last_used={k.last_used_at}")

# Walk every Key under one Project
cursor = None
while True:
    page = kova.keys.list(project_id="01J8X9Y2Z3K4M5N6P7Q8R9PRJ1", limit=100, after=cursor)
    for k in page.data:
        print(f"{k.last4}  {k.name:24}  {k.status}  {k.mode}")
    if not page.has_more:
        break
    cursor = page.next_cursor
```
