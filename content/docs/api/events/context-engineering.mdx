---
title: Context engineering events
description: GET /v1/ks/events/context-engineering — pull manifest, segment, artifact, compaction, and ContextBudget admission events under ks.context.*.
---

Read context engineering events — the lifecycle of every context-engineering primitive: [manifests](/docs/concepts/context/context-manifests), [segments](/docs/concepts/context/context-segments), [artifacts](/docs/concepts/context/context-artifacts), [compaction records](/docs/concepts/context/compaction-records), and [ContextBudget](/docs/concepts/policy/context-budgets) admission decisions.

```http
GET /v1/ks/events/context-engineering
Authorization: Bearer ks_...
```

## Authorization scope

Bearer key bound to a `(tenant, project, environment)` triple. Returns events whose `tenant_id` matches the resolved AuthContext.

## Query parameters

| Parameter | Type | Default | Meaning |
|---|---|---|---|
| `after` | string \| null | `null` | Opaque cursor — return events strictly after this position. |
| `before` | string \| null | `null` | Opaque cursor — return events strictly before this position. |
| `limit` | integer | `100` | Maximum events per page. Capped at `500`. |
| `run_id` | ULID | `null` | Restrict to events for a single Run. |
| `step_id` | ULID | `null` | Restrict to events for a single Step. |
| `model_call_id` | ULID | `null` | Restrict to events for a single ModelCall. |
| `context_manifest_id` | ULID | `null` | Restrict to events for a single ContextManifest. |
| `context_budget_id` | ULID | `null` | Restrict to events for a single ContextBudget. |
| `decision` | string | `null` | Filter ContextBudget events by admission decision (`WARN`, `OFFLOAD_REQUIRED`, `COMPACT_REQUIRED`, `DEGRADE`, `HARD_STOP`, `ALLOW`). |
| `event_type[]` | string | `null` | Filter to specific subjects. Repeat for multiple values. |
| `since` | RFC 3339 | `null` | Include only events with `timestamp >= since`. |
| `until` | RFC 3339 | `null` | Include only events with `timestamp < until`. |

## Event types

Manifests:

| `event_type` | Trigger |
|---|---|
| `ks.context.manifest.created` | A new ContextManifest row is inserted. |
| `ks.context.manifest.admitted` | Admission settles with `ALLOW` or `WARN`. The admitted manifest is the one that proceeds to dispatch. |

Segments:

| `event_type` | Trigger |
|---|---|
| `ks.context.segment.created` | A ContextSegment row is inserted. |
| `ks.context.segment.evicted` | A segment is removed under retention or compaction. |

Artifacts:

| `event_type` | Trigger |
|---|---|
| `ks.context.artifact.created` | A ContextArtifact row is inserted (object-storage upload completed). |
| `ks.context.artifact.offloaded` | An inline segment was rewritten as an artifact reference under `OFFLOAD_REQUIRED`. |

Compaction:

| `event_type` | Trigger |
|---|---|
| `ks.context.compaction.started` | A CompactionRecord is opened. |
| `ks.context.compaction.completed` | Compaction finished; output segments are usable. |

ContextBudget:

| `event_type` | Trigger |
|---|---|
| `ks.context.budget.warning` | Admission produced `WARN`. |
| `ks.context.budget.offload_required` | Admission produced `OFFLOAD_REQUIRED`. |
| `ks.context.budget.compact_required` | Admission produced `COMPACT_REQUIRED`. |
| `ks.context.budget.degraded` | Admission produced `DEGRADE`. |
| `ks.context.budget.hard_stop` | Admission produced `HARD_STOP`. |

## Request

```http
GET /v1/ks/events/context-engineering?run_id=01J8X9Y2Z3K4M5N6P7Q8R9S0T1
Authorization: Bearer ks_...
```

```bash
curl -s -G "https://api.kovaserve.com/v1/ks/events/context-engineering" \
  -H "Authorization: Bearer ks_..." \
  --data-urlencode "run_id=01J8X9Y2Z3K4M5N6P7Q8R9S0T1"
```

```python
import requests

r = requests.get(
    "https://api.kovaserve.com/v1/ks/events/context-engineering",
    headers={"Authorization": f"Bearer {api_key}"},
    params={"run_id": "01J8X9Y2Z3K4M5N6P7Q8R9S0T1"},
)
events = r.json()["data"]
```

### `ks.context.budget.*`

All ContextBudget events share a payload shape:

```json
{
  "context_budget_id": "01J8X9Y2Z3K4M5N6P7Q8R9CB01",
  "decision": "OFFLOAD_REQUIRED",
  "dimension": "active_tokens",
  "limit": 16384,
  "consumed": 18420,
  "consumed_pct": 112.43,
  "transformation_planned": "artifact_offload"
}
```

`decision` matches the event type's suffix. `dimension` ∈ `active_tokens`, `stable_tokens`, `volatile_tokens`, `tool_result_tokens`, `artifact_bytes`. `transformation_planned` is the action the [admission](/docs/concepts/policy/admission) service is about to apply.

## Identity fields

`context_manifest_id` is populated on every event tied to a specific manifest. `run_id`, `step_id`, and `model_call_id` are populated for events fired in the per-call hot path (manifest assembly, admission, compaction triggered during the call). Standalone artifact events may have `run_id` and `model_call_id` `null` when the write is not call-bound.

## Sequence around a single ModelCall

```
ks.context.segment.created                  (per inbound segment)
ks.context.manifest.created                 (the Global Context Manager produces the initial manifest)
[optional] ks.context.budget.offload_required
ks.context.artifact.offloaded               (one per offloaded segment)
[optional] ks.context.budget.compact_required
ks.context.compaction.started
ks.context.compaction.completed
ks.context.manifest.admitted                (final admitted shape)
```

The sequence may include zero, one, or multiple admission round-trips depending on transformations applied. The `ks.context.manifest.admitted` event always lands last for the call, with the final decision (`ALLOW` or `WARN`).

## Status codes

| Status | Meaning |
|---|---|
| `200 OK` | Page returned. Empty `data` is valid. |
| `400 invalid_request` | Malformed cursor, both `after` and `before` set, `limit` outside `[1, 500]`, unknown `event_type` or `decision`. |
| `401 invalid_api_key` | Bearer token missing, malformed, or revoked. |
| `403 forbidden` | Filter targets a Run, Manifest, or ContextBudget outside the key's scope. |
| `429 rate_limit_exceeded` | Per-key request rate limit reached. |

## Common usage

**Context timeline UI.** Per-Run timeline of segment composition, manifest evolution, and admission decisions. Filter by `run_id` and merge by `timestamp`.

**Compaction lineage view.** Filter `event_type[]=ks.context.compaction.completed` and walk by `payload.source_manifest_id` to draw the lineage tree.

**ContextBudget dashboards.** Aggregate `ks.context.budget.*` decisions per Project to surface budget-tuning needs.

**Cache reuse audit.** Join `ks.context.manifest.admitted.payload.stable_context_id` against the [state events](/docs/api/events/state) `kv.state.accessed.payload.stable_context_id` to verify reuse predictions.

**Offload accounting.** Sum `ks.context.artifact.offloaded.payload.byte_count` per Run to attribute offload usage against [ContextBudget](/docs/concepts/policy/context-budgets) limits.
