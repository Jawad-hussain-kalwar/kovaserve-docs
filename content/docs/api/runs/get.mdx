---
title: Get a Run
description: GET /v1/ks/runs/{id} — fetch a Run with its Steps and ModelCalls embedded in canonical timeline order.
---

`GET /v1/ks/runs/{id}` returns a single [Run](/docs/concepts/run-hierarchy/runs) with its [Steps](/docs/concepts/run-hierarchy/steps) and each Step's [ModelCalls](/docs/concepts/run-hierarchy/model-calls) embedded inline. The shape is the canonical timeline payload — clients reconstruct a Run's full execution trace from a single response without N+1 fetches.

The Run is resolved against the API key's Tenant. Cross-tenant fetches return `404 Not Found` — the same response as a genuinely missing Run — so the surface leaks no information about Runs the caller cannot see.

## Endpoint

```
GET /v1/ks/runs/{run_id}
```

| Property | Value |
|---|---|
| Auth | `Authorization: Bearer ks_...` — see [Authentication](/docs/api/using-the-api/authentication). |
| Path parameter | `run_id` — bare 26-character Crockford ULID. |
| Errors | Native envelope — see [Errors](/docs/api/using-the-api/errors). |

## Response — `200 OK`

The body is the Run row with `steps` populated. Each Step carries its own `model_calls` array. Steps are returned in `ordinal` order (the order they were admitted under the Run); ModelCalls inside a Step are returned in `created_at` order.

### Top-level fields

| Field | Type | Meaning |
|---|---|---|
| `id` | string | Run ID. |
| `tenant_id` / `project_id` | string | Owning tenancy pair. |
| `session_id` | string \| null | Bound Session, when set. |
| `policy_scope_id` | string | Resolved PolicyScope. |
| `budget_id` / `context_budget_id` | string \| null | Attached Budget and ContextBudget. |
| `branch_from` | string \| null | Parent Run when this Run is a branch. |
| `status` | string | Current state in the [Run state machine](/docs/concepts/run-hierarchy/run-state-machine). |
| `model` | string \| null | Default model. |
| `external_run_id` / `external_runtime` | string \| null | External-runtime correlation fields. |
| `step_count` | integer | Number of admitted Steps. |
| `created_at` / `started_at` / `completed_at` | string \| null | Lifecycle timestamps. |
| `steps` | array | Embedded Steps. See [Steps](/docs/concepts/run-hierarchy/steps). |

### Embedded Step fields

| Field | Type | Meaning |
|---|---|---|
| `id` | string | Step ID. |
| `run_id` | string | Owning Run. |
| `step_type` | string | One of the canonical [step types](/docs/concepts/run-hierarchy/step-types). |
| `status` | string | One of the values in the [Step state machine](/docs/concepts/run-hierarchy/step-state-machine). |
| `ordinal` | integer | The Step's position under the Run. Monotonic, gap-free. |
| `parent_step_id` | string \| null | The Step that admitted this one, when nested (e.g., a `TOOL_EXECUTE` admitted by a preceding `MODEL_CALL`). |
| `created_at` / `started_at` / `completed_at` | string \| null | Step lifecycle timestamps. |
| `metadata` | object | Free-form Step-type-specific metadata. |
| `model_calls` | array | Embedded ModelCalls for the Step. Empty for non-`MODEL_CALL` Step types. |

### Embedded ModelCall fields

| Field | Type | Meaning |
|---|---|---|
| `id` / `step_id` / `run_id` | string | Identity and parents. |
| `model` | string | Model that actually executed. May differ from the Run's default when the Budget is in `DEGRADED_ALLOWED`. |
| `node_id` | string | Node that served the call. |
| `prompt_tokens` / `completion_tokens` | integer | Real token counts from the substrate. |
| `ttft_ms` / `total_ms` | integer | Time-to-first-token and total wall-clock latency in milliseconds. |
| `cache_hit_ratio` | number | Fraction of the prompt that hit a warm KV cache, on `[0.0, 1.0]`. |
| `cost_cents` | string | Cost in cents as a decimal string with 4 fractional digits. See [Cost headers](/docs/api/using-the-api/cost-headers). |
| `stable_prefix_id` | string \| null | Hex digest of the cache-aligned prefix. |
| `context_manifest_id` | string | Anchored [ContextManifest](/docs/concepts/context/context-manifests). |
| `created_at` | string | ISO-8601 timestamp. |

## Status codes

| Status | When |
|---|---|
| `200 OK` | The Run exists and is visible to the caller. |
| `401 Unauthorized` | Missing or invalid bearer token. Codes: `missing_token`, `invalid_token`, `token_revoked`. |
| `404 Not Found` | The Run does not exist *or* belongs to another Tenant. Code: `run_not_found`. |

## Examples

```bash
curl https://api.kovaserve.com/v1/ks/runs/01J8X9Y2Z3K4M5N6P7Q8R9S0T1 \
  -H "Authorization: Bearer ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b"
```

```python
from kovaserve import KovaClient

kova = KovaClient(api_key="ks_live_a7f3c2b8d4e1f0925a1b6c3d4e5f6a7b")

run = kova.runs.get("01J8X9Y2Z3K4M5N6P7Q8R9S0T1")

print(f"status: {run.status}, steps: {run.step_count}")
for step in run.steps:
    print(f"  [{step.ordinal}] {step.step_type} → {step.status}")
    for mc in step.model_calls:
        print(f"      {mc.model} on {mc.node_id}: ${mc.cost_cents}, cache {mc.cache_hit_ratio:.0%}")
```
