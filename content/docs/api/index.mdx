---

KovaServe's HTTP API is the public interface to the [control plane](/docs/concepts/layers/control-plane). It exposes two coexistent families on the same base URL:

- **OpenAI-compatible inference** — `/v1/chat/completions` and `/v1/models`. Drop-in usage from any OpenAI SDK works without code changes; the `kovaserve` extension envelope opts into native features when you want them.
- **Native control plane** — `/v1/ks/*` for [Runs](/docs/concepts/run-hierarchy/runs), [Sessions](/docs/concepts/continuity/sessions), [Checkpoints](/docs/concepts/recovery/checkpoints), context engineering, budgets, policy, metering, tenancy, runtime, events, and cluster operations.

Start with [Using the API](/docs/api/using-the-api) for the cross-cutting conventions every endpoint shares — base URL, authentication, errors, pagination, idempotency, IDs, cost headers, and streaming. Then jump to the resource you need.
